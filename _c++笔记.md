3. | **模式**     | **语法示例**          | **权限说明**             | **常见用途**                     |
   | ------------ | --------------------- | ------------------------ | -------------------------------- |
   | **常量指针** | `const int *p`        | **内容不可变**，地址可变 | 函数参数（保护输入数据不被修改） |
   | **指针常量** | `int * const p`       | 内容可变，**地址不可变** | 硬件寄存器映射、引用模拟         |
   | **只读指针** | `const int * const p` | **内容、地址均不可变**   | 极高安全要求的只读配置           |

4. `int (*p)(int, int)` 拆解开看：
   1. **`(\*p)`**：说明 `p` 是一个指针。
   2. **右边的 `(int, int)`**：说明这个指针指向的是一个函数，且该函数接收两个整型参数。
   3. **左边的 `int`**：说明该函数返回一个整型值。

5. 由内向外，先右后左

   `char* (*p[5])(int)` 像剥洋葱一样剥开：

   ## 1. 拆解步骤（剥洋葱法）

   1. **核心是 `p[5]`**： 先看中间，`p` 后面跟着 `[5]`。
      - **结论**：`p` 是一个**拥有 5 个元素的数组**。
   2. **往左看 `\*`**： 数组名 `p` 被 `*` 修饰了。
      - **结论**：这个数组里的每个元素都是一个**指针**。（到这里你已经拆出“指针数组”了！）
   3. **往右看 `(int)`**： 跳出括号看右边，发现有参数列表。
      - **结论**：这些指针指向的是**函数**，且函数接收一个 `int` 参数。
   4. **往左看 `char\*`**： 看最前面的返回类型。
      - **结论**：这些函数执行完后，返回的是一个 **`char\*`（字符串地址）**。

   ------

   ## 2. 最终大白话翻译

   `char* (*p[5])(int)` 的意思是：

   > **`p` 是一个数组，里面装了 5 个“快捷键”（函数指针）。当你按下其中一个快捷键并传给它一个整数时，它会运行一个特定的程序，最后给你吐出一个字符串的地址。**

6. **普通类型（如 `int`）**：它就像一个**实物盒子**，里面装的就是具体数值（比如 `10`）。

   **指针类型（如 `int\*`）**：它就像一张**小纸条**，上面写着那个盒子的具体位置（比如 `0x0012FF`）。

7. 函数，函数的静态变量,static，有记忆性，能反复利用

   ## 📝 C 语言笔记：`strcspn` 函数与输入清洗

   #### 1. 函数原型

   需包含头文件：`#include <string.h>`

   C

   ```
   size_t strcspn(const char *str1, const char *str2);
   ```

   - **功能**：从 `str1` 开头往后数，计算**不包含** `str2` 中任何字符的连续长度。
   - **直观理解**：它在问“在撞见 `str2` 里的倒霉孩子之前，前面有多少个干净字符？”

   ------

   #### 2. 核心应用：清洗 `fgets` 的换行符

   这是你代码中最关键的一行：

   C

   ```
   newNode->name[strcspn(newNode->name, "\n")] = '\0';
   ```

   **为什么执行这一行？**

   - **现象**：`fgets` 会把用户按下的回车键（`\n`）也存进字符串。
   - **后果**：打印时会导致不该换行的地方强制换行，破坏 UI 整齐。
   - **逻辑**：
     1. `strcspn(newNode->name, "\n")` 找到第一个 `\n` 出现的**下标**。
     2. 如果没找到 `\n`（输入太长被截断了），它会返回末尾 `\0` 的下标。
     3. 直接将该位置赋值为 `\0`，实现“精准切除回车”或“原地踏步（安全）”。

   ------

   #### 3. `strcspn` 常用场景对比

   | **目标**         | **代码示例**               | **返回值 (假设 s="SYSU-2026")**            |
   | ---------------- | -------------------------- | ------------------------------------------ |
   | **找特定符号**   | `strcspn(s, "-")`          | **4** (跳过 SYSU，在下标 4 撞见 `-`)       |
   | **找任意数字**   | `strcspn(s, "0123456789")` | **5** (跳过 SYSU-，在下标 5 撞见 `2`)      |
   | **检测非法字符** | `strcspn(s, " #*")`        | **9** (数完整个字符串都没撞见空格、# 或 *) |

   ------

   #### 4. 避坑指南：与 `strlen` 的区别

   - `strlen(s)`：计算到 `\0` 为止的总长度。
   - `strcspn(s, "\n")`：计算到 **第一个换行符** 为止的长度。
   - **结论**：在处理输入时，`strcspn` 比 `strlen` 更智能，因为它能精准定位你想删掉的那个回车。

9. **二级指针 `\**headPtr` 的真相：**
   1. **修改权**：在函数内部修改 `main` 中 `head` 指针的**指向**。
   2. **物理逻辑**：`head` 是存地址的变量，`headPtr` 是存 `head` 地址的变量。
   3. **应用场景**：仅在需要改变“链表头是谁”时（如空表插入、头部删除）才是必须的。

10. 在**尾插法**中，新人后面是没人的（`NULL`）。如果你让第一个人和新人先连，比如写成 `*headPtr = newNode`，那么原本链表后面的 B、C、D 同学就全部**丢失**了。

11. **固定指针（`head`）**：永远守住入口，不许动。

    **移动指针（`temp`/`curr`）**：负责跑腿、查数、寻找

12. **修改 vs 移动：**
    - `ptr = ptr->next;` $\rightarrow$ **移动指针**（像在看书，翻到下一页）。
    - `ptr->next = newNode;` $\rightarrow$ **修改结构**（像在改书，把当前页的结尾撕了，贴上一张新的）。

13. ## 野指针”与“悬挂指针”的笔记整理

    你可以把这段话写进你的 PARA 笔记，这是 SYSU 考试中关于内存管理的高频考点：

    - **本质**：当一块内存被 `free` 掉后，所有指向它的指针都变成了**“悬挂指针” (Dangling Pointer)**。
    - **危害**：它们就像指向不存在房间的钥匙。你以为能开门，结果开门后可能是万丈深渊（崩溃），也可能是别人的房间（修改了不该改的数据）。
    - **最佳实践**：
      1. `free` 之前，确保有其他指针接管了后续的链表。
      2. `free` 之后，如果该指针还在作用域内，习惯性地给它赋值为 `NULL`（例如：`curr = NULL;`）。

### 3.6

1. # 📚 C 语言底层逻辑：声明顺序与自引用结构体

   ## 1. 核心铁律：先声明，后使用

   C 语言编译器是**单向扫描**（从上往下）的。任何标识符（变量名、函数名、结构体名、绰号）在被使用前，必须先在编译器的“名单”里登记。

   ------

   ## 2. 自引用结构体的“悖论”解谜

   为什么 `struct Student` 还没定义完，内部就能使用 `struct Student *next`？

   ## 💡 核心机制：前向引用 (Forward Reference)

   当编译器读到 `struct Student {` 时，它已经完成了两件事：

   1. **登记标签**：在符号表中记录了 `struct Student` 这个类型标签。
   2. **指针豁免**：编译器知道 `next` 是一个**指针**。

   > **底层真相**：
   >
   > 指针的大小是固定的（64 位系统为 8 字节）。编译器此时不需要知道 `struct Student` 的具体成员和总大小，它只需要在内存布局中预留出 8 个字节的“坑位”来存放地址即可。

   ------

   ## 3. `typedef` 的先后顺序陷阱

   为什么在结构体内不能直接使用 `Node *next`？

   C

   ```
   typedef struct Student {
       int id;
       struct Student *next; // ✅ 合法：struct Student 已在第一行登记
       // Node *next;        // ❌ 非法：Node 这个绰号在最后一行才生效
   } Node;
   ```

   - **逻辑拆解**：
     1. `struct Student` 是**本名**，在第一行就露脸了。
     2. `Node` 是**绰号**，只有在大括号彻底结束（第 5 行）后，编译器才完成“绰号登记”。
     3. 在大括号内部（第 3 行），编译器还不认识 `Node`。

   ------

   ## 4. 解决“先有鸡还是先有蛋”：前向声明

   当两个结构体**相互引用**时（例如：男人有妻子，女人有丈夫），必须手动打破顺序僵局。

   ## ❌ 错误尝试：

   C

   ```
   struct Man { struct Woman *wife; }; // 报错：不认识 Woman
   struct Woman { struct Man *husband; };
   ```

   ## ✅ 正确做法：前向声明 (Forward Declaration)

   我们在最前面先跟编译器“打个招呼”：

   C

   ```
   struct Woman; // 【前向声明】告诉编译器：后面有个类型叫 Woman，先有个印象
   
   struct Man {
       struct Woman *wife; // 现在合法了，编译器知道这是一个指向 Woman 的指针
   };
   
   struct Woman {
       struct Man *husband;
   };
   ```
